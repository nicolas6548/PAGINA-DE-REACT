  3. VIRTUAL DOM

    4. Primero vamos a ver que es el DOM

    El DOM (Document Object Model es:

    • Una repersentacion en árbol de la estructura de un documento HTML/XML
    • Una interfaz de programación que permite manippular documentos web
    • Actualizado directamente por el navegador cuando hay cambios

      • Algunas caracteristicas del DOM real son:

    • Es la reppresentación directa de lo que ves en el navegador (pero en forma de arbol, donde ves desde la JERARQUIA de las etiquetas del documento web, desde las etiquetas padres, pasando por las etiquetas hijas, nietas y todas las que tenga la pagina web
    • Cada cambio en la pagina web, provoca un REFLOW (recalculo de posicioones) y repaint (redibujado), un pequeño ejemplo es en FACEBOOK, al hacer un comentario en una publicacion, se hace un REFLOW y REPAINT de todo el documento HTML en el cual se carga el comentario que se ingreso (lo que en documentos extensos y maquinas con pocos recursos eso hacia muy lenta la carga de datos constante)

    Pero en FACEBOOK dijeron: ¿NO SE PUEDE MEJORAR ESTO?

    Y crearon el VIRTUAL DOM

    ¿QUE ES EL VIRTUAL DOM?

    • Una abstraccion ligera del DOM real
    • Una copia en memoria de la representacion del DOM

      • Y ya me imagino sus caras jajajajjaja, el DOM VIRTUAL se almacena en memoria toda la pagina web (en si no toda, sino el sector que se esta viendo en ese preciso momento), vamos a dar un ejemplo sencillo para que se entienda

      GOOGLE MAPS, al abrilo, nos muestra una porcion del mapa donde estamos actualmente, en la parte superior (en algunos casos) nos muestra algunas opciones como: RESTAURANTES, MUSEOS, INFORMACION UTIL, ETC, si nosotros queremos ver que museos hay cerca de donde estamos, no se refrezca toda la pagina para mostrarnos los museos, sino que se compara la informacion que pedimos con el DOM VIRTUAL y solo nos agrega la informacion solicitada, todo lo que ya estaba cargado lo deja y solo carga lo que necesitamos, de esa forma es mas rapido y consume menos recursos que el DOM al cargar todo cada ves que haciamos una consulta o busqueda

  4. DATA BLINDING UNIDIRECCIONAL

    5. ¿QUE SIGNIFICA?

    El data binding unidireccional (flujo de datos unidireccional) significa que los datos fluyen en una sola dirección en la aplicacion:

    • De los componentes padres a los componentes hijos
    • Nunca en la dirección inversa (los hijos no pueden modificar directamente las propiedades que reciben

    Caracteristicas clave

    • FLUJO PREDECIBLE: Los datos solo van en una dirección, haciendo más fácil entender cómo cambia el estado
    • PROPIEDADES DE SOLO LECTURA: Los componentes hijos reciben datos pero no los modifican directamente
    • ACTUALIZACION CONTROLADA: Los cambios solo ocurren cuando el estado del componente padre cambia
